# Contributing

## Overview

Contributions are welcome but not required. This project maintains a lightweight contribution model.

## Pull Requests

Pull requests are optional. If you wish to contribute:

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request

## Intellectual Property

By contributing to this project, you agree that all contributions become the property of DatumStudios. You grant DatumStudios a perpetual, worldwide, non-exclusive license to use, modify, and distribute your contributions.

## Code Style

- Follow existing code style and conventions
- Keep changes minimal and focused
- Ensure code compiles without errors or warnings

## Review Process

All contributions are subject to review by the project maintainers. There is no guarantee that contributions will be accepted.
